# views 디렉터리 역할

상태 스토어를 구독해 화면을 조합하는 컨테이너 레이어다.

## 책임 구분
- 프로그래머 관점: `App`, `Layout`, `Workbench` 등 뷰 컴포넌트가 스토어에서 데이터를 가져와 `components`에 전달하고, 사용자 이벤트를 Action으로 디스패치한다. 필요 시 간단한 라우팅 또는 레이아웃 관리도 담당한다.
- 학생 관점: 화면의 큰 틀을 담당하는 코드가 있다. 여기서 데이터를 받아서 어떤 컴포넌트를 보여줄지 결정하고, 사용자가 버튼을 누르면 상태 변경을 요청한다.

## 시스템에서의 필요성
- 컨테이너가 없으면 UI 조각이 서로 연결되지 않아 전체 화면이 동작하지 않는다.
- 상태 변경을 한 방향으로 흘려보내 단방향 데이터 흐름을 지킨다.

## 다른 폴더와의 상호작용
- `src/state`: 스토어에서 상태를 구독하고, Action 디스패치를 호출한다.
- `src/components`: 프레젠터 컴포넌트에 데이터를 주입해 화면을 구성한다.
- `src/services`: 필요 시 서비스 함수를 호출하여 체크포인트 저장/복원을 트리거한다.
- `web/tests`: 사용자 여정을 시뮬레이션할 때 뷰 레벨에서 이벤트를 발생시켜 결과를 검증한다.

## 구현 가이드
- 상태 구독 해제를 명확히 관리해 메모리 누수를 방지한다.
- 라우터 도입 시에도 Action 디스패치 → Reducer → 상태 갱신 → 렌더 흐름을 유지한다.

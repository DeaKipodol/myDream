# lib 디렉터리 역할

도메인에 종속되지 않는 공통 유틸리티 모듈을 제공한다.

## 책임 구분
- 프로그래머 관점: DOM 헬퍼, 이벤트 버스, 로거, assert 함수 등 재사용 가능한 도구를 구현한다. 다른 폴더에서 동일한 기능을 반복 작성하지 않도록 한다.
- 학생 관점: 여러 곳에서 똑같이 쓰는 도우미 코드를 한 폴더에 모아 필요할 때 가져다 쓰는 곳이다. 기능 자체는 상담 로직과 직접 관련이 없다.

## 시스템에서의 필요성
- 반복되는 보일러플레이트를 줄여 유지보수 시간을 줄인다.
- 로깅과 검증 유틸을 통해 디버깅과 오류 탐지를 빠르게 한다.

## 다른 폴더와의 상호작용
- `src/components`와 `src/views`: DOM 조작과 이벤트 유틸을 가져와 UI 렌더를 단순화한다.
- `src/state`: 로거와 assert를 활용해 액션 흐름을 추적한다.
- `src/services`: 이벤트 버스나 로거를 활용해 비동기 처리 상태를 알린다.
- `web/tests`: 유틸 함수 자체를 검증하거나 테스트 헬퍼로 재사용한다.

## 구현 가이드
- 유틸은 도메인 정보나 전역 상태를 직접 참조하지 않는다.
- 공통 패턴이 반복될 때만 추가하고, 남용을 피하기 위해 문서화한다.

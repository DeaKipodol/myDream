# state 디렉터리 역할

전역 상태 스토어, 액션, 리듀서를 정의해 단방향 데이터 흐름을 형성한다.

## 책임 구분
- 프로그래머 관점: `store.js`에서 상태와 구독/알림 로직을 만들고, `actions.js`와 `reducers.js`에서 이벤트 정의와 상태 전이를 구현한다. 옵저버 패턴과 미들웨어 확장 포인트를 제공한다.
- 학생 관점: 현재 상담 단계, 선택한 노드, 체크포인트 목록 같은 값을 한곳에 저장하고 바꾸는 코드가 있다. 화면이 바뀌려면 여기서 상태가 업데이트되어야 한다.

## 시스템에서의 필요성
- 상태가 없으면 TreeView, Breadcrumbs, Chat이 서로 다른 정보를 보여주게 된다.
- 단일 스토어를 두면 어떤 이벤트가 어떤 변화를 만들었는지 추적하기 쉽다.

## 다른 폴더와의 상호작용
- `src/views`: 상태를 구독해 화면을 만들고, 사용자 이벤트를 액션으로 디스패치한다.
- `src/algorithms`: 리듀서가 경로 계산을 위해 알고리즘 함수를 호출한다.
- `src/services`: 체크포인트 저장/복원 결과를 상태로 반영하거나, 상태를 전달해 서비스를 호출한다.
- `web/tests`: 액션과 리듀서 단위 테스트를 수행해 회귀를 방지한다.

## 구현 가이드
- 상태 객체는 얕은 복사를 유지해 변경 추적을 명확히 한다.
- 액션 타입과 페이로드 스키마를 주석으로 명시하여 디버깅과 협업을 지원한다.

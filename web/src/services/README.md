# services 디렉터리 역할

체크포인트 저장, 선반 관리 등 부작용을 담당하는 서비스 계층이다.

## 책임 구분
- 프로그래머 관점: Memento 패턴을 활용해 경로 스냅샷을 저장하고 복원하며, shelves 관리, 알림 로직을 실행한다. 퍼사드 모듈을 통해 UI 계층에 단일 API를 제공한다.
- 학생 관점: 상담 중 나중에 돌아오기 위해 경로를 저장하거나, 여러 시나리오를 비교하려면 이 폴더의 코드가 필요하다. 화면은 요청만 하고, 실제 저장과 복원은 여기서 진행한다.

## 시스템에서의 필요성
- 체크포인트 기능이 없으면 잘못된 선택에서 빠르게 돌아갈 수 없다.
- 선반 관리가 없으면 여러 시도 결과를 모아 비교할 수 없다.

## 다른 폴더와의 상호작용
- `src/storage`: 드라이버를 통해 데이터를 실제로 읽고 쓴다.
- `src/state`: 서비스 결과를 상태로 반영하거나, 상태 데이터를 받아 외부 저장소에 기록한다.
- `src/views`: 사용자가 버튼을 누르면 서비스 함수를 호출하고, 완료 후 상태 업데이트를 요청한다.
- `web/tests`: 서비스의 저장/복원 시나리오를 테스트해 데이터 손실을 방지한다.

## 구현 가이드
- Promise 반환 시 에러를 명확히 throw하고, 호출 측이 처리할 수 있도록 메시지를 포함한다.
- 퍼사드에 추가되는 함수는 문서화하고, 내부 서비스 간 결합을 최소화한다.

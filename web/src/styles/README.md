# styles 디렉터리 역할

정적 데모의 시각 언어와 레이아웃을 정의하는 CSS 자산을 관리한다.

## 책임 구분
- 프로그래머 관점: 디자인 토큰, 리셋, 컴포넌트 스타일, 유틸리티 클래스를 분리해 유지보수성을 높인다. CSS 커스텀 프로퍼티를 통해 테마 변경에 대응한다.
- 학생 관점: 화면의 색상, 글꼴, 여백을 정하고 정리하는 파일이 모여 있다. 여기 값을 바꾸면 화면 분위기가 달라진다.

## 시스템에서의 필요성
- 명확한 스타일 구조가 없으면 컴포넌트가 일관성 없이 보이고 협업이 어렵다.
- 토큰을 사용하면 접근성 기준(명도 대비 등)을 지키기 쉽다.

## 다른 폴더와의 상호작용
- `src/components`와 `src/views`: 클래스 이름과 토큰 값을 가져와 UI에 적용한다.
- `public/index.html`: 초기 스타일시트를 로드한다.
- `web/tests`: 시각 회귀 테스트나 체크리스트에서 스타일 토큰을 비교한다.

## 구현 가이드
- CSS 파일은 명확한 역할별로 나누고, 중복 선언을 피한다.
- 접근성 요구가 변경되면 토큰 파일을 우선 업데이트하고, 필요한 경우 컴포넌트 스타일을 조정한다.

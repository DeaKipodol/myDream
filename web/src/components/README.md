# components 디렉터리 역할

사용자가 보는 화면 요소를 렌더링하는 프레젠터 컴포넌트를 모은다.

## 책임 구분
- 프로그래머 관점: 입력 props를 받아 DOM을 생성하거나 갱신하는 순수 렌더 함수 또는 클래스다. 상태 변경 로직은 포함하지 않으며, 접근성 속성과 이벤트 훅만 정의한다.
- 학생 관점: TreeView, Breadcrumbs, Chat 영역처럼 화면에 보이는 조각을 만드는 코드가 있다. 어떤 데이터를 보여줄지 결정하기만 하고, 데이터를 바꾸는 일은 하지 않는다.

## 시스템에서의 필요성
- 뷰가 없으면 사용자가 상담 흐름을 확인할 수 없다. 이 폴더가 있어야 `views`가 계산된 데이터를 읽어 화면을 구성한다.
- 순수 렌더로 유지하면 디자인 수정이나 스냅샷 테스트가 쉬워진다.

## 다른 폴더와의 상호작용
- `src/views`: 컨테이너가 상태를 읽어 컴포넌트에 props를 전달하고, 사용자 이벤트를 받아 Action으로 변환한다.
- `src/styles`: 공통 토큰과 컴포넌트별 클래스 정의를 가져와 스타일을 적용한다.
- `web/tests`: 시각 변화가 예상대로인지 스냅샷 테스트를 수행한다.

## 구현 가이드
- props 인터페이스를 `models`에서 정의한 타입과 맞춰 문서화한다.
- 이벤트 핸들러는 직접 상태를 조작하지 말고, 콜백을 통해 상위 컨테이너에 알린다.

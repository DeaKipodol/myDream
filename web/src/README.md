# src 디렉터리 역할

정적 웹 앱의 실행 로직을 모듈 단위로 배치해 단방향 데이터 흐름과 알고리즘, UI, 서비스 책임을 분리한다.

## 책임 구분
- 프로그래머 관점: 엔트리 `app.js`가 상태 스토어를 초기화하고, `views`를 렌더링한다. 폴더별 모듈을 통해 Flux-lite 스토어, LCA 계산, 체크포인트 서비스를 구성한다.
- 학생 관점: 상담사가 어떤 경로를 선택할지 계산하고, 화면에 표시하고, 저장해 두는 코드는 모두 여기 있다. 필요한 기능을 찾으면 해당 폴더의 모듈을 수정하면 된다.

## 시스템에서의 필요성
- 상담 흐름을 계산하거나 UI를 갱신하는 로직이 없으면 `public/index.html`은 빈 틀만 남는다.
- 각 책임을 모듈로 나누면 버그 위치를 빠르게 찾고, 새 기능을 추가할 때 다른 부분을 건드리지 않는다.

## 다른 폴더와의 상호작용
- `web/public`: `index.html`이 `src/app.js`와 스타일 파일을 로드한다.
- `web/tests`: 테스트 코드가 `src` 모듈을 import하여 알고리즘과 상태 변화를 검증한다.
- `docs/architecture`: 여기서 정의한 구조와 패턴을 구현할 때 참고하는 기준 문서다.

## 하위 구조 요약
- `app.js`: 스토어 초기화, 루트 렌더, 이벤트 바인딩.
- 폴더별 세부 책임은 각 README에 정리되어 있으며, 부작용은 `services/`와 `storage/`에서만 허용한다.

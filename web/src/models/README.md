# models 디렉터리 역할

도메인 객체 스키마와 타입 가드를 정의해 데이터 구조를 한곳에 모은다.

## 책임 구분
- 프로그래머 관점: 노드, 체크포인트, 메시지, 사용자 입력 등 데이터 구조를 코드로 정의하고, 검증 유틸리티를 제공한다. 다른 모듈은 이 정의를 import해 일관된 타입을 사용한다.
- 학생 관점: 트리를 표현하려면 어떤 정보가 필요하고, 체크포인트를 저장하려면 어떤 값이 있어야 하는지 목록을 정리한 곳이다. 규칙을 어기면 애플리케이션이 제대로 움직이지 않는다.

## 시스템에서의 필요성
- 구조 정의가 흩어져 있으면 데이터가 섞이고 버그를 찾기 어렵다.
- 모델을 통해 입력 검증을 하면 잘못된 데이터가 상태나 서비스로 흘러가는 것을 막을 수 있다.

## 다른 폴더와의 상호작용
- `src/algorithms`: 노드 구조를 참고해 경로 계산을 수행한다.
- `src/state`: 상태 객체의 형태를 정의하고, 액션 페이로드 검증에 활용한다.
- `src/services`: 체크포인트 저장 시 모델을 통해 직렬화/역직렬화 규칙을 따른다.
- `web/tests`: 모델을 기반으로 테스트 데이터 픽스처를 생성한다.

## 구현 가이드
- 스키마 변경 시 관련 테스트와 문서를 함께 업데이트한다.
- 입력 검증 함수는 오류 메시지를 명확히 반환해 디버깅을 돕는다.

# mocks 디렉터리 역할

데모와 테스트에 사용하는 샘플 데이터를 모아둔다.

## 책임 구분
- 프로그래머 관점: 상담 트리, 메시지, 체크포인트, 로그 등의 JSON 또는 JS 객체를 정의해 알고리즘과 UI를 검증한다. 실제 백엔드가 없을 때도 일정한 입력을 제공한다.
- 학생 관점: 아직 서버를 만들지 않았어도 화면이 어떻게 동작하는지 확인하기 위해 임시 데이터를 넣어두는 폴더다. 여기 데이터를 바꾸면 데모에서 보이는 내용이 달라진다.

## 시스템에서의 필요성
- 샘플 데이터가 없으면 TreeView와 Chat 화면이 비어 있어서 기능을 설명하기 어렵다.
- 일관된 픽스처가 있어야 테스트 결과를 비교할 수 있다.

## 다른 폴더와의 상호작용
- `src/services`: 초기 로딩 시 목업 데이터를 가져와 상태를 채운다.
- `src/state`: 초기 상태를 만들 때 목업 데이터를 사용한다.
- `web/tests`: 테스트에서 동일한 입력을 사용해 결과를 재현한다.

## 구현 가이드
- 데이터 계약을 주석으로 남기고, 실제 백엔드 스키마와 달라질 경우 업데이트한다.
- 대규모 데이터는 별도 파일로 분리해 유지보수 부담을 줄인다.

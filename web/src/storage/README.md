# storage 디렉터리 역할

데이터를 실제로 저장하거나 불러오는 드라이버를 추상화한다.

## 책임 구분
- 프로그래머 관점: 메모리 기반 드라이버와 `localStorage` 드라이버를 구현하고, 공통 인터페이스(`get`, `set`, `remove`, `list`)를 제공한다. 서비스 계층이 저장 방식에 의존하지 않도록 한다.
- 학생 관점: 페이지를 새로 고치거나 나중에 다시 열어도 상담 기록을 남기고 싶다면 이 폴더의 코드가 데이터를 대신 보관해 준다.

## 시스템에서의 필요성
- 체크포인트나 선반 데이터를 저장하지 않으면 복원 기능이 동작하지 않는다.
- 드라이버를 분리하면 나중에 IndexedDB, 원격 API 등으로 쉽게 교체할 수 있다.

## 다른 폴더와의 상호작용
- `src/services`: 저장/복원 기능을 실행할 때 드라이버를 호출한다.
- `web/tests`: 드라이버가 약속한 인터페이스를 준수하는지 테스트한다.
- `src/models`: 저장되는 데이터 구조를 모델 정의와 일치시키기 위해 import한다.

## 구현 가이드
- 브라우저 환경 의존이 있는 드라이버는 존재 여부를 체크하고 예외 메시지를 제공한다.
- 영속 저장 시 버전 정보를 함께 기록해 데이터 구조 변경에 대비한다.

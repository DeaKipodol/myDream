# tests 디렉터리 역할

정적 앱의 동작을 검증하는 테스트 코드와 체크리스트를 모은다.

## 책임 구분
- 프로그래머 관점: 단위 테스트, 스냅샷 테스트, 수동 점검 시나리오를 관리한다. Jest/Playwright 등의 자동화 도구를 도입할 위치를 예약한다.
- 학생 관점: 우리가 만든 화면이 제대로 작동하는지 단계별로 확인하는 절차를 적어둔 곳이다. 여기 있는 순서대로 실행하면 버그를 빠르게 찾을 수 있다.

## 시스템에서의 필요성
- 테스트가 없으면 LCA 계산이나 상태 전환이 깨졌을 때 즉시 알 수 없다.
- 체크리스트를 통해 데모 전 검증 시간을 단축하고 신뢰도를 높인다.

## 다른 폴더와의 상호작용
- `src`: 테스트가 모듈을 import하여 알고리즘, 상태, 컴포넌트를 검증한다.
- `web/public`: 초기 로딩 테스트에서 `index.html`을 열어 화면이 뜨는지 확인한다.
- `docs`: 테스트 계획과 품질 게이트 기준을 문서로 정리하고, 실행 결과를 기록한다.

## 구현 가이드
- 테스트 데이터는 `src/mocks`에서 가져오거나 별도 픽스처를 정의한다.
- 자동화 도구를 추가할 때는 의존성을 최소화하고, 브라우저 호환성을 확인한다.

# public 디렉터리 역할

정적 웹 애플리케이션이 처음 로드될 때 브라우저가 직접 요청하는 파일을 관리한다.

## 책임 구분
- 프로그래머 관점: `index.html`, 파비콘, 메타 태그, manifest 등을 배치한다. ES 모듈 스크립트와 스타일시트 링크가 여기서 정의되므로, 경로가 바뀌면 `src` 모듈 위치도 함께 조정해야 한다.
- 학생 관점: 사용자가 주소를 입력하면 가장 먼저 열리는 파일이 이 폴더에 있다. HTML 파일에서 자바스크립트와 CSS를 불러와 상담 화면을 보여준다.

## 시스템에서의 필요성
- `index.html`이 없으면 브라우저가 무엇을 렌더링할지 몰라 빈 화면이 나타난다.
- 메타 정보는 공유 썸네일, 접근성, PWA 설정 등 추가 요구를 충족한다.

## 다른 폴더와의 상호작용
- `web/src`: `index.html`의 `<script type="module">`와 `<link rel="stylesheet">`가 `src/app.js`, `src/styles/...`를 불러온다. 경로 변경 시 HTML과 `src` 둘 다 수정한다.
- `web/tests`: 수동 테스트 중 첫 단계가 `public/index.html`을 열어 초기 화면 로딩을 확인하는 것이다.

## 작성 가이드
- 정적 호스팅이 캐싱하므로 파일명 변경 시 배포 계획을 업데이트한다.
- 후속 단계에서 백엔드 통신이 필요하면 `<script>`에서 API 엔드포인트를 주입하기보다 `src/services`에 설정을 전달한다.

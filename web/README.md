 # web 디렉터리 개요

AI 고민상담 대화 분기/체크포인트/경로 전환을 눈으로 확인할 정적 웹 애플리케이션의 루트다. 정적 호스팅 환경(GitHub Pages, S3 등)에 그대로 올릴 파일을 이 디렉터리에서 관리한다.

## 역할 요약
- 프로그래머 관점: 정적 배포 단위를 구성하는 최상위 패키지다. `public`, `src`, `tests` 하위 모듈이 서로 의존하며 프론트엔드 UI, 상태 관리, 검증 흐름을 완성한다.
- 학생 관점: 여기 있는 파일만 준비하면 브라우저에서 상담 시나리오 시연 화면을 바로 띄울 수 있다. 폴더 세 곳이 함께 움직여서 “어떤 화면을 보여줄지”, “어떻게 반응할지”, “문제가 없는지”를 나눠 담당한다.

## 하위 폴더 책임
- `public/`
	- 프로그래머 관점: 진입점 HTML과 브라우저가 직접 읽는 정적 자산을 제공한다. 배포 시 CDN이 가장 먼저 접근하는 위치이므로 경로 구조가 고정된다.
	- 학생 관점: 사용자가 페이지를 열면 가장 먼저 읽는 파일이 여기 있다. 이 안의 HTML이 `src`의 스크립트와 스타일을 불러와 화면을 만든다.
- `src/`
	- 프로그래머 관점: 컴포넌트, 상태 스토어, 알고리즘, 서비스가 모듈 단위로 배치된다. Flux-lite 흐름과 LCA 계산 로직이 이 폴더에서 실행된다.
	- 학생 관점: 상담 트리 계산과 화면 반응을 실제로 만드는 코드가 모여 있다. 여기 내용을 바꾸면 화면 동작이 달라진다.
- `tests/`
	- 프로그래머 관점: 단위 테스트, 스냅샷 검증, 수동 점검 시나리오를 정리한다. `src` 모듈을 가져와 동작을 검증한다.
	- 학생 관점: 우리가 만든 화면이 예상대로 움직이는지 시험하는 절차를 담는다. 목록대로 따라 하면 오류를 빨리 찾을 수 있다.

## 다른 폴더와의 상호작용
- `docs/`: 규칙과 설계를 명세한 문서로, 이 디렉터리 구조가 준수해야 할 기준을 제공한다. 직접 파일 I/O는 없지만 업데이트 시 이 문서를 기준으로 검토한다.
- `web/public` ↔ `web/src`: `public/index.html`이 `src/app.js`와 `src/styles` 자원을 로드한다. 경로 변경 시 두 폴더를 동시에 수정해야 한다.
- `web/tests` ↔ `web/src`: 테스트 코드가 `src` 모듈을 import하여 LCA 계산, 상태 전환, 렌더 함수가 의도대로 작동하는지 확인한다.

## 운영 시 유의 사항
- 빌드 도구를 사용하지 않으므로 모든 import는 상대 경로로 유지하고, ES 모듈 구문을 브라우저가 직접 실행할 수 있도록 작성한다.
- 정적 호스팅에서 404가 발생하지 않도록 폴더 구조와 파일명을 문서에 정의된 대로 유지한다.
- 백엔드 연계를 추가할 때는 `src/services`와 `src/storage`에 어댑터를 작성하고, `public`의 엔트리 HTML에는 추가 스크립트를 과도하게 삽입하지 않는다.

# 스모크 테스트 체크리스트

## 1. 목적
- 주요 화면과 상호작용이 손상되지 않았는지 빠르게 확인한다.
- 체크포인트 기능 통합 시 회귀 여부를 조기에 감지한다.
- 모든 구현 작업 종료 후 본 문서와 `implementation-progress.md`를 함께 갱신한다.

## 2. 사전 준비
- macOS Safari 또는 Chrome 최신 버전.
- `web/public/index.html`을 브라우저로 직접 열거나 정적 서버로 서비스한다.
- 새 탭을 열어 캐시되지 않은 상태에서 테스트를 시작한다.

## 3. 기본 렌더링
- [ ] 페이지 로드시 헤더, 탭, 대화 패널, 분기 리스트 패널, 체크포인트 패널이 모두 보인다.
- [ ] 초기 상담 경로 메시지가 대화 패널에 3단계 이상 표시되고 단계별로 구분된다.
- [ ] 분기 리스트에 루트 노드의 직접 하위 분기만 표시된다.
- [ ] 대화 메시지가 AI는 좌측, 사용자는 우측에 채팅 스타일로 정렬된다.

## 4. 경로 전환
- [ ] 분기를 클릭하면 활성 분기가 강조되고 단계 선택기에 해당 경로의 단계가 표시된다.
- [ ] 단계 선택기의 동그라미를 클릭하면 해당 단계의 대화 섹션으로 이동한다.
- [ ] 서로 다른 두 분기를 연속 클릭했을 때 강조와 대화 내용이 일관되게 갱신된다.
- [ ] 이미 활성화된 노드를 다시 클릭해도 불필요한 깜박임이나 오류가 발생하지 않는다.

## 5. 체크포인트
- [ ] "체크포인트 저장" 버튼 클릭 시 목록에 새 항목이 추가된다.
- [ ] 새 항목이 생성되면 상태 메시지에 최근 작업 내용이 표시된다.
- [ ] 체크포인트 항목 선택 시 저장 시점의 상담 경로가 복원된다.
- [ ] 복원 직후 대화 패널, 단계 선택기, 분기 리스트 강조가 동시에 업데이트된다.
- [ ] 동일 노드를 여러 번 저장해도 목록이 모두 유지되고 고유 식별 정보로 구분된다.

## 6. 회귀 주의 사항
- 경로 계산 알고리즘(LCA, `buildActivePath`) 수정 후 모든 클릭 시나리오가 정상인지 확인한다.
- 체크포인트 복원 중 예외 발생 시 UI가 정상 상태로 롤백되는지 확인한다.
- 빈 목록에서 복원을 시도할 수 없도록 버튼 상태나 메시지가 방지 로직을 제공하는지 검토한다.
- 동일 경로를 반복 저장했을 때 상태 메시지가 적절히 안내되는지 확인한다.
- 단계 선택기와 분기 리스트가 UI 업데이트 사이클에서 동기화되는지 검증한다.

## 7. 테스트 기록
- 스모크 테스트를 수행한 날짜, 브라우저, 발견 이슈를 `docs/status/progress/implementation-progress.md`에 추가한다.

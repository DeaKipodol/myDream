## 2. 프로토타입 MVP 범위 확인 요청

**질문**: 프로토타입 단계에서 다음 중 어디까지 구현해야 하나요?

### 옵션 A: 최소 기능 (프론트엔드만)
- 트리 구조 시각화 (TreeView)
- 대화 패널 (ChatView)
- 브레드크럼
- **데이터**: 하드코딩된 샘플 데이터
- **경로 전환**: 프론트엔드 내에서만 동작
- **예상 소요**: 1-2일

**장점**: 빠른 검증, 핵심 UX 테스트 가능
**단점**: 백엔드 없음, AI 없음

### 옵션 B: 프론트 + 백엔드 기본 (API 연동)
- 옵션 A + 백엔드 API
- 경로 전환 API (POST /conversations/{id}/switch)
- 노드 조회 API
- **데이터**: 메모리 기반 (실제 DB 없음)
- **예상 소요**: 3-4일

**장점**: 프론트-백 통합 검증, 실제 API 흐름 확인
**단점**: AI 없음, 체크포인트 없음

### 옵션 C: 프론트 + 백엔드 + 체크포인트
- 옵션 B + 체크포인트 저장/복원 API
- 보존 스레드 기본 기능
- **예상 소요**: 5-6일

**장점**: 핵심 기능 대부분 구현
**단점**: AI 없음

### 옵션 D: 풀 MVP (AI 통합 포함)
- 옵션 C + AI 응답 생성
- 실제 대화 가능
- **예상 소요**: 7-10일

**장점**: 완전한 동작 데모
**단점**: 시간 소요

## 3. 우선순위 확인 요청

**질문**: 다음 중 어떤 순서로 개발하면 좋을까요?

### 제안 A: UX 우선
1. 프론트엔드 (트리뷰 + 대화패널)
2. 경로 전환 로직 (프론트 내)
3. 백엔드 API
4. 체크포인트
5. AI 통합

**이유**: 사용자 경험을 먼저 검증하고, 백엔드는 나중에 연동

### 제안 B: 아키텍처 우선
1. 백엔드 핵심 (LCA, 경로 전환)
2. API 엔드포인트
3. 프론트엔드 (API 연동)
4. 체크포인트
5. AI 통합

**이유**: 핵심 알고리즘을 먼저 검증하고, UI는 나중에 붙임

### 제안 C: 수직 슬라이스 (추천)
1. **슬라이스 1**: 트리뷰 + 샘플데이터 + 경로전환(프론트만) (1일)
2. **슬라이스 2**: 백엔드 API + 프론트 연동 (2일)
3. **슬라이스 3**: 체크포인트 기능 (1-2일)
4. **슬라이스 4**: AI 통합 (2-3일)

**이유**: 각 단계에서 동작하는 결과물이 나옴. 피드백 빠름

## 4. 아키텍처 설계자 협의 필요 사항

다음 사항을 아키텍처 설계자에게 질문 예정:

1. **프로토타입 파일 구조**:
   - 설계서의 구조를 그대로 따를까요?
   - 아니면 `prototype/` 폴더에 간소화된 구조로 시작할까요?

2. **프론트엔드 기술 스택**:
   - Vanilla JS로 시작? (설계서 권장)
   - 아니면 React/Vue로 바로 시작?

3. **백엔드 데이터 저장**:
   - 프로토타입 단계: 메모리? JSON 파일? SQLite?
   - 언제 MongoDB로 전환?
